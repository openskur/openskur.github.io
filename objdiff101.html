Objdiff 101

The objdiff file is JSON file with information about variances in a SKUR diff. The downloaded file by default will be named diff.objdiff.json. The SKUR engineering team frequently iterates over the generation of this file with more and more useful information. Your file may look a little different depending of the version of objdiff you have.

Below is an example of an objdiff file with 1 variance. { "version":"10.4.0.9", "clusterDensityUnits":"points/cm^3", "objectList": [ {"id":"100", "dm":0.333, "pointsInObj":900, "seenPoints":288, "closePoints":11, "consideredCount":148, "missing":0, "clusterDensity":0.9999, "centroid":[3,5,6], "modelSister":[8,4,6], "scanBB":[1,2,3,4,5,6] }, ] }

Let's step through some of the fields and explain what they mean.

"version":"10.4.0.9"

This is probably self-explanatory, but this field tells us the version of objdiff.

"objectList":[]

The objectList is an array of variances.

Let's take a closer look at the fields of the individual variance.

"id":"100"

The id of the variance.

"dm":0.333

The max variance.

"pointsInObj":900

The number of points in the scanned object. This is a derived number based on the model.

"seenPoints":288

The number of points actually seen by the scanner.

"closePoints":11

The number of points that are potentially displaced and determined close by the SKUR intelligence engine.

"consideredCount":148

The number of points that were considered for displacement.

"missing":0

The number of missing points.

"centroid":[3,5,6]

The x,y,z location of the variance.

"modelSister":[8,4,6]

The x,y,z location of a point on the model from which a variance corresponds to.

"scanBB":[8,4,6,2,3,4]

An array of 6 points that define the bounding box of the scanned object: [xmin, ymin, zmin, xmax, ymax, zmax]
